// technical architecture
// 技术架构
Six-layer engineering stack.
六层工程架构。

Built around policy-to-control translation, multi-depth evidence collection, and privacy-preserving deployment — not just a database in a UI wrapper.

围绕法规到控制项翻译、多深度证据收集和隐私保护部署构建，而不只是套了个 UI 的数据库。

L1
Regulatory Knowledge Layer
监管知识层
Regulations, standards, benchmarks as machine-readable objects: jurisdiction · trigger · obligation · evidence · control · severity
法规、标准与基准测试结构化为机器可读对象：辖区 · 触发器 · 义务 · 证据 · 控制项 · 严重性
CORE
L2
Applicability Engine
适用性引擎
Answers "which rules apply and why" — versioned, traceable obligation map per product × market
回答"哪些规则适用及原因"——为每个产品×市场组合输出版本化、可追溯的义务地图
ENGINE
L3
Assurance Ingestion Layer
合规信号采集层
Unified intake for questionnaires, docs, screenshots, API probes, benchmark outputs, local scan findings
统一接收问卷、文档、截图、API 探针、benchmark 输出与本地扫描结果
LAYER
L4
Black-box Testing Layer
黑盒测试层
Evaluates model behavior via API or test env — no source code. Runs safety, bias, transparency, and logging checks
通过 API 或测试环境评估模型行为，无需源代码。执行安全性、偏见、透明度和日志检查
LAYER
L5
Private Scan Layer
私有扫描层
Self-hosted scanner, VPC deploy, CI/CD plugin — code never leaves your environment. Bring the engine to the code.
自托管扫描器、VPC 部署和 CI/CD 插件——代码永远不离开你的环境。把引擎带到代码那里。
ENGINE
L6
Evidence Graph
证据图谱
Obligations, controls, test results, docs, and gaps in an explainable audit chain — single source of truth for launch decisions
将义务、控制项、测试结果、文档和缺口组织为可解释审计链——上线决策的单一可信来源
CORE
